# 场景关系证据模型

## 两层关系

本 Skill 严格区分两层：

1. **底层数据证据**：字段指纹重合、文件哈希、显式文件引用、表头结构和带定位的文档句子。
2. **场景语义关系**：参与方、输入、活动、对象、规则、决策、状态、系统和输出之间的业务关系。

底层证据不是最终图。具体值只用于证明“这两份材料可关联”，不能成为节点，也不能自动推出时间顺序或因果关系。

## 证据卡

每张卡包含稳定 `E-...` ID、类型、强度、说明、来源文件、定位和有界事实。

| 类型 | 含义 | 可证明范围 |
|---|---|---|
| `scenario_goal` | 用户提供的业务目标 | 词汇和范围，不能独立证明关系 |
| `goal_relation_statement` | 场景描述中明确的要求、条件或结果 | 用户明确规定的业务关系 |
| `file_structure` | 文件类型、大小、表数量和解析状态 | 材料存在及覆盖 |
| `table_schema` | 表头按业务角色分组 | 材料角色和概念存在 |
| `table_process_signal` | 同一表中多种业务角色共现 | 结构联系，不能独立确定时序 |
| `field_relationship` | 不暴露原值的精确指纹联系 | 跨材料追溯、引用或依赖 |
| `table_relation_statement` | 小型高信号表中带单元格定位的规则或条件短句 | 短句明确表达的约束、条件或结果 |
| `document_relation_statement` | 带页码/段落/OCR 行号的关系句 | 句子明确表达的方向、条件或结果 |
| `material_topic_alignment` | 文本主题与另一份材料文件名语义对齐 | 主题相关性，只能作为补强证据 |
| `document_outline` | 有界标题与大纲 | 文档角色和上下文 |

关系证据规则：

- `references`、`depends_on`、`joins_with` 可以由精确 `field_relationship` 直接支撑。
- `feeds` 可由精确跨材料关系与材料角色共同支撑；`governs`、`derives` 优先要求场景描述或明确关系句。
- `triggers`、`precedes`、`branches_to`、`returns_to` 优先要求明确的场景、表格或文档关系句；没有直接关系句时，至少需要流程结构卡与另一张独立结构/直接证据共同支撑。
- `produces`、`consumes`、`transforms`、`updates`、`governed_by`、`performed_by` 需要明确关系句，或多张相互支持的结构/直接证据。
- 场景目标和文件元数据不能独立证明一条业务边。

## 场景图约束

- 节点类型：`trigger`、`actor`、`input`、`activity`、`object`、`rule`、`decision`、`state`、`system`、`output`。
- 边类型另含宏观数据关系 `feeds`、`joins_with`、`governs`、`derives`；优先用这些类型形成数据底图。
- `main_chain` 是兼容字段，语义为主数据路径；至少三个节点，起点为触发、输入、参与方或系统，终点为输出、状态或对象。
- 主数据路径相邻节点间必须有同方向的数据流或推导边。
- 最终图最多 10 个节点、14 条边、3 个分支；建议保持在 5-8 个节点。
- 分支首边必须是 `branches_to`，并从已经挂接的节点出发。
- 所有节点在无向视角下必须属于同一连通分量。
- 图中每个纳入文件必须出现在节点、关系或分支引用的证据来源中。

## 覆盖与不确定性

`complete` 表示：

- 底层探针完成所有可解析表的候选列扫描；
- 文档按声明的字符预算提取了关系证据；
- 文件均被纳入图或有证据地排除；
- 最终图通过连通性、主支链和证据门槛验证。

它不表示系统能够证明原始材料中没有任何未明说的隐含关系。不能达到门槛的猜想不进入最终边，应在报告中列为待业务确认项。

## 大数据性能边界

- 从小型、高信息密度材料开始，只定向扫描大表的候选列。
- XLSX 优先使用 Rust Calamine 驱动的 `fastexcel`；DuckDB 和 `openpyxl` 是后备路径。
- CSV/TSV、JSONL、SQLite 流式读取；Parquet 用列式读取。
- 不持久化全量 distinct 值，不拼接大表整行，不为每个单元格执行实体正则。
- 种子、bottom-k 列摘要、每值命中数、文档字符数、每文件关系句和总证据卡数均有上限。
- 每 50,000 行写断点；`partial` 后用原命令恢复。
- OCR 原文只存在临时文件，进入 Agent 上下文的只有短证据卡。
