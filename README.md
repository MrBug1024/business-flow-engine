# 业务能力蒸馏平台

把人类沉淀在「历史数据 + 规则说明 + 历史结果」里的业务经验，**蒸馏**成可被 AI 大模型调用的
Skill（任务流），让 AI 在特定业务场景下具备完成该业务的能力。

> 核心比喻：**模型蒸馏**。蒸馏（训练）阶段做的事，绝不在使用（推理）阶段重做——
> 表关系与规则在蒸馏阶段固化，运行时直接读取，不重新分析。

## 设计红线（见 `docs/IMPLEMENTATION.md` 第二章）

1. **零业务硬编码**：框架代码场景无关，所有表名/字段/规则/输出格式只存在于知识包（数据），不出现在逻辑里。
2. **AI 绝不接触原始数据行**：AI 推理层只读元数据报告（schema/基数/样本/规则摘要），全量数据只在采样层与执行层流动。
3. **输出复刻历史结果格式**：默认产出文件（Excel/Word/MD…），结构与历史结果一致。
4. **防御「有数据却产出 0 条」**：执行每步记录行数，骤降为 0 即告警。
5. **全程人工可介入**：不确定即生成问答交互，绝不瞎猜。
6. **验收由人工判断**：用户上传测试数据 + 提需求 → 执行 → 看结果是否想要。
7. **表角色由用户指定**，绝不靠文件名猜。
8. **表头智能识别**，不默认第一行当表头。

## 架构（三层分离）

```
原始数据 → [元数据采样层 纯Python] → 元数据报告(摘要) → [AI推理层 MiniMax 只读摘要]
                                                          ↓ 执行步骤链(逻辑)
原始数据 ───────────────────────────────────────→ [验证执行层 纯Python] → 差异摘要 ↺ 回AI迭代
                                                          ↓
                              知识包(ER模型/规则库/任务流/输出模板) → [运行时: AI大脑 + 通用Skill执行引擎]
```

## 目录

```
backend/
  knowledge/   知识包数据结构(Pydantic) 与 按场景隔离的存储
  metadata/    第一层：表头智能识别 / 文件解析 / schema采样 / 规则·结果解析
  reasoning/   第二层：ER建模 / 规则库 / 三层反推(语义→符号回归→拟合) / 问答交互 / 蒸馏编排
  verification/第三层：全量执行 + 差异摘要
  runtime/     通用执行引擎 / DSL安全编译 / 节点handler / 输出复刻 / 运行时AI大脑
  api/         FastAPI 路由
  app.py       入口
frontend/      纯 HTML + 原生 JS + mermaid（只读图谱，无 Vue / 无构建）
data/scenarios/<id>/  每个场景独立：知识包 + uploads/outputs/models/embeddings/versions
```

## 运行

```bash
pip install -r requirements.txt
cp .env.example .env      # 填入 MiniMax API Key；不填则走启发式模式
python run.py             # http://127.0.0.1:8000
```

打开浏览器：新建场景 → 上传素材并标注角色 → 运行蒸馏（处理问答交互）→ 查看只读图谱
→ 上传测试数据提需求执行 → 人工验收发布。

## 通用性自检

同一套未修改的框架代码可跑通完全不同的场景（医保审计 / 销售分析 / 质检报告…）。
若换场景需要改框架代码，即说明存在硬编码——这是项目成败的核心标准（见文档 10.3）。
